Moduł III. Działanie przeglądarki

Protokół HTTP

HTTP - Hypertext Transfer Protocol
HTML = HyperText Markup Language
http = protokół do transportowania htmla
HTTPS - Hypertext Transfer Protocol Secure

URL

URL - Uniform Resource Locator Schemat działanie przeglądarki:
- wpisany adres w pasku
- przeglądarka szuka serwera [DNS]
- po znalezieniu serwera odpytuje go o index (jeśli podamy sam adres serwera) - index zależy od konfiguracji serwera (w przypadku stron statycznych może być to index.html)
- serwer wysyła do przeglądarki zawartość html
- przeglądarka wczytuje html i szuka wszystkich dodatkowych zasobów: style, skrypty, obrazki itp. i dla każdego z zasobów wykonuje zapytanie
- przeglądarka parsuje i renderuje stronę

Kody odpowiedzi HTTP

1xx - hold on
200 - OK - 2xx - here you go
30x - Szukasz w złym miejscu - przekieruję Cię w dobre (bo wiem gdzie) - 3xx - go away
404 - Nie mam tego czego potrzebujesz - 4xx - you fucked up
50x - Błąd wewnętrzny serwera - 5xx - I fucked up

root (/)

Jest istotną częścią ścieżek do zasobów np. podczas publikacji na serwerze. Podczas uruchamiania z dysku lokalnego ładowana zawartość jest bez "/".
Dzieje się tak ponieważ brak "/" z przodu ścieżki czyni adres do zasobu relatywnym (przeglądarka szuka pliku w aktualnym "katalogu")

Position, float

Warianty pozycji:
- position: static; (statyczna) - rozmieszczenie uzależnione od przeglądarki, nie trzeba definiować - tryb domyślny
- position: relative; - pozwala stworzyć przestrzeń względem której możemy rozmieszczać inne elementy
- position: absolute; (absolutne pozycjonowanie wg elementu relative) - korzysta z ustawionej przestrzeni relative, według niej następuje rozmieszczanie, w tym celu można używać up-right-bottom-left (liczone od krawędzi) - romieszczenie nie zmieni się po zmianie rozmiarów elementu do którego się odnosimy
- z-index; (kolejność wyświetlania warstw) - pozwala na priorytet wyświetlania elementów
- position: fixed; odpowiada za przyklejenie elementu do viewportu (okna przeglądarki) np. w aplikacjach mobilnych do przyklejenia przycisku menu
- position: sticky; możliwość przyklejenia, doczepienia elementu, który działa wewnątrz rodzica (w przeciwnieństwie do fixed) np. do przyklejania nagłówków tekstów - nie zadziała, gdy zdefiniowany overflow oraz brakuje parametru kierunku np. top
- float element jest opływany przez treść znajdujący się po nim, nie jest zalecane stosowanie, nie używany do budowania layoutu!!!
- clear czyści floaty, powoduje, że treść wpada poniżej

Typy zmiennych JS

JavaScript to język nietypowany(zmienne mogą mieć różne typy). Typy zmiennych (typy proste - primitives):
- String - tekst, otczany ' lub " (zalecane ')
- Number - liczba
- Bigint - duża liczba, większa niż 2^53
- boolean - wartość logiczna true lub false, do logiki i sprawdzania warunków
- null - wartość celowo niezdefiniowana
- undefined - brak definicji
- *symbol - tworzy unikalne opakowanie, jest ale się nie używa^^

Sprawdzanie typu zmiennej poprzez polecenie typeof <zmienna>, wartości zwracane dla poszczegłonych typów: - String -> 'string'
- Number -> 'number'
- boolean -> 'boolean'
- undefined -> 'undefined'
- null -> 'object'

Immutability tzw. niemutowalność, czyli że nie możemy zmieniać wartości primitive values w JS, zmieniane jest natomiast połączenie między nimi a zmiennymi.

Moduł IV. BEM

BEM - Block Element Modifier

Specjalny model nazywania klas css, który wprowadza do projektu porządek:
- Block - nadrzędnu element;
- Element - częśc bloku(nie musi być bezpośrednim dzieckiem, ale musi się odnosić do elementu);
- Modifier - modyfikator - jeśli element występuje w kilku wariantach tworzymy dodatkową klasę;

Sposób zapisu BEM: block__element--modifier. Block jest zawsze, element dodajemy, jeżeli występuje element, a modifier różnicuje elementy. W przypadku BEM:
- szukamy elementów na stronie i dzielimy je na bloki
- myślimy o elementach składowych / robimy analizę-selekcję - szukamy elementów powtarzalnych i unikalnych
- tworzymy dwie klasy jeśli element ma modyfikator (w modyfikatorze zmieniamy tylko unikalne dla niego wartości, nie przepisujemy rzeczy z klasy bazowej) Przykłady użycia:

Moduł IV. DOM

DOM - Document Object Model

DOM - Obiektowy Model Dokumentów, widoczny z poziomu DevTools. Jest to swego rodzaju pomost pomiędzy JS a HTML. DOM to złożony w przeglądarce HTML, który może być modyfikowany przez JS. UWAGA: Plik HTML ani źródło strony to nie jest DOM.
Selektor JS (działa tak samo jak w CSS):
Funkcja document.querySelector('.selector') odpowiednik $('.selector') w języku jQuery. Funkja querySelector zwraca node(węzeł) z HTML'a, a selektor zapisywany zgodnie z CSS.
Stosując BEM elementy powinny być oznaczane w taki sposób (dotyczy elementów odnoszących się do JS):
block__element--js
Tego elementu nie stylujemy w pliku css, bo jest on wykonywany przez skrypt i po to żeby było bardziej czytelnie(ułatwia identyfikację, pomaga usuwać problemy).
Użycie defer pozwala na ukrycie zawartości węzła, przykład poniżej:
<script defer src="index_notes.js"></script>
Przykład użycia - dla nagłówka h1:
<h1 class="page-header__heading page-header__heading--js">Notatki ze szkolenia WTF: Co Ten Frontend. Nagłówek tekstowy header - <h1></h1>
Składnia operacji JS na elemencie:
nazwaZmiennejQuerySelector.innerHTML = 'treść' - modyfikacja wyświetlonej treści
Przykład operacji JS:
const header = document.querySelector('.page-header__heading--js');
header.innerHTML = 'Hej';
